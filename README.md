# 俄罗斯方块 (Tetris)

一款基于 HTML5 Canvas 开发的经典俄罗斯方块游戏，具有精美的3D视觉效果和流畅的游戏体验。

---

## 📋 程序概述

本项目是一个纯前端实现的俄罗斯方块游戏，无需后端支持，可直接在浏览器中运行。游戏采用现代 Web 技术栈开发，支持桌面端和移动端访问。

### 技术栈

| 技术              | 版本 | 说明                 |
| ----------------- | ---- | -------------------- |
| HTML5             | -    | 页面结构和语义化标签 |
| CSS3              | -    | 样式设计和动画效果   |
| JavaScript (ES6+) | -    | 游戏逻辑实现         |
| Canvas API        | -    | 图形渲染引擎         |

### 文件结构

```
├── elsfk.html    # 主游戏文件（包含HTML、CSS、JS）
├── README.md     # 项目说明文档
└── elsfk-functions.txt  # 函数调用关系文档
```

## ✨ 功能特性

### 🎮 核心游戏功能

- **七种经典方块**：I、O、T、S、Z、J、L 七种标准方块
- **7-Bag 随机系统**：保证每种方块均匀出现，避免连续重复
- **锁定延迟机制**：方块触底后给予玩家调整时间
- **幽灵方块提示**：显示方块落点预览，帮助玩家决策

### ⚡ 操作特性

- **DAS 移动系统**：支持按住方向键连续移动
- **软降加速**：按住下键加速下落，每格得1分
- **硬降功能**：空格键直接落到底部，每格得2分
- **踢墙旋转**：支持多种偏移位置的旋转尝试

### 🎨 视觉效果

- **3D 方块渲染**：带高光和阴影的立体方块效果
- **消行动画**：消除行时的闪烁特效
- **分数弹跳动画**：得分时的视觉反馈
- **响应式设计**：自适应桌面端和移动端

### 📊 计分系统

- **等级系统**：每消除10行升一级
- **速度递增**：等级越高，下落速度越快
- **得分倍率**：消除行数越多，得分倍率越高

| 消除行数 | 基础得分 |
| -------- | -------- |
| 1行      | 100分    |
| 2行      | 300分    |
| 3行      | 500分    |
| 4行      | 800分    |

> 实际得分 = 基础得分 × 当前等级

## 🎯 操作指南

### 键盘操作

| 按键  | 功能               |
| ----- | ------------------ |
| ← →   | 左右移动方块       |
| ↑     | 旋转方块（顺时针） |
| ↓     | 软降（加速下落）   |
| Space | 硬降（直接落底）   |
| P     | 暂停/恢复游戏      |

### 移动端操作

| 虚拟按键 | 功能             |
| -------- | ---------------- |
| ↻        | 旋转方块         |
| ← →      | 左右移动         |
| ↓        | 软降（加速下落） |
| 硬降     | 直接落底         |
| 暂停     | 暂停/继续游戏    |

- **触摸游戏区域**：游戏结束时点击屏幕重新开始
- **布局特点**：信息栏位于游戏画布上方，虚拟按键位于底部

### 游戏规则

1. 方块从顶部随机生成并下落
2. 使用方向键移动和旋转方块
3. 方块落地后固定，自动生成新方块
4. 填满一行自动消除并得分
5. 方块堆积到顶部则游戏结束
6. 按空格键重新开始游戏

## ❓ 常见问题解答

### Q1：游戏无法运行？

**可能原因**：

- 浏览器版本过低
- JavaScript 被禁用
- 文件路径包含中文或特殊字符

**解决方案**：

1. 升级浏览器到最新版本
2. 确保浏览器启用了 JavaScript
3. 将文件移动到纯英文路径下

### Q2：键盘操作无响应？

**可能原因**：

- 焦点不在游戏页面上
- 页面被其他元素遮挡
- 浏览器阻止了键盘事件

**解决方案**：

1. 点击游戏区域激活焦点
2. 检查页面是否有弹窗或遮挡元素
3. 刷新页面重新加载

### Q3：游戏卡顿或延迟？

**可能原因**：

- 浏览器标签页过多
- 设备性能较低
- 网络连接不稳定（在线部署时）

**解决方案**：

1. 关闭其他不必要的标签页
2. 降低游戏区域分辨率（修改 CELL_SIZE）
3. 切换到本地运行模式

### Q4：移动端无法操作？

**可能原因**：

- 触摸事件未正确触发
- 虚拟键盘遮挡游戏区域
- 屏幕尺寸过小

**解决方案**：

1. 确保浏览器支持触摸事件
2. 关闭虚拟键盘或使用外接键盘
3. 横屏模式获得更好体验

### Q5：如何保存游戏进度？

**当前状态**：本游戏暂不支持本地存储功能

**解决方案**：

- 如需保存进度，可考虑添加 localStorage 支持
- 或部署到支持后端存储的服务器

## 📝 开发说明

### 代码结构

```javascript
// 主要模块划分
1. DOM 元素引用        // 获取页面元素
2. 游戏常量定义        // 尺寸、颜色、形状
3. 游戏状态管理        // board、currentPiece、score等
4. 辅助函数           // 矩阵操作、边界计算
5. 游戏板操作         // 创建、检测、锁定、消行
6. 方块管理           // 生成、移动、旋转、硬降
7. 渲染函数           // Canvas绘制、动画效果
8. 游戏逻辑           // 重力、锁定延迟、计分
9. 输入处理           // 键盘、触摸事件
10. 初始化和事件绑定   // 启动游戏
```

### 核心函数说明

| 函数名             | 功能说明                     |
| ------------------ | ---------------------------- |
| `init()`           | 初始化游戏，创建初始方块     |
| `gameLoop()`       | 游戏主循环，处理帧更新       |
| `spawnPiece()`     | 生成新方块（从预览或随机袋） |
| `lockAndProcess()` | 锁定方块并处理消行           |
| `tryMove()`        | 尝试移动方块（含碰撞检测）   |
| `tryRotate()`      | 尝试旋转方块（含踢墙检测）   |
| `render()`         | 渲染所有游戏元素             |
